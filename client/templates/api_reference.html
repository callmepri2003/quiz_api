Reference

    • This API has 4 main endpoints
    • /api/new_quiz/
    • /api/submit_quiz/
    • /api/students_info/
    • The new_quiz endpoint takes the username directly from the basic auth.
    • For example: quiz = requests.get(current_host + new_quiz_endpoint, auth=(username, password))
    • The JSON it returns is structured as follows:
      • data = {
    'quiz_attempt_id': new_quiz_attempt.id,
    'quiz': {
        'q1': SerialisedQuestionObject
        'q2': ...
        ...
    }
}
    • The submit_quiz endpoint takes JSON structured as such:
    • The quiz attempt id is simply passed from the initial request, submitted_answers is a dictionary with the key as the QuizContent ID, and the value as the submitted answers. 
      • {
    'quiz_attempt_id': 245,
    'submitted_answers': {
        "64" : [1,2,3],
        ...
        ...

    }
}
    • And it returns the following JSON
      • { 'score': percentageRepresentedByInteger }

  • The students_info endpoint simply returns a dictionary of serialised student objects with their id as the keys.
